Design candidates · Gradient editor v2

Цвет по % — четыре направления

Текущий редактор работает (после фикса BLE chunking и lock-fix первого stop'а), но интерфейс плотный и нет ни пресетов, ни плавного градиента. Ниже — четыре варианта переработки, от минимальной правки до полной интеграции. Каждый пример рабочий — можно поменять цвет, перетащить, попробовать пресет.

Stack: HTML/CSS/JS · Azeret Mono · Onest Aesthetic: operator console Theme: dark
01
Refined operator EVOLUTION
Тот же редактор, но в три раза просторнее. Первый stop залочен на 0%. Маркеры на полосе сразу видны как «булавки» — таскаете цвет за пин, остальные параметры в табличке снизу.
Live gradient strip PCT 0 ↔ 100
saved

Что нового

  • Первый stop залочен на 0% — цвет редактируется, % серый и недоступный
  • Полоса градиента в 3× выше, под ней — шкала 0/10/20…/100
  • Стопы видны как пины на полосе — наглядно где они
  • Hover на row → подсветка соответствующего диапазона

Когда брать

  • Если хочется минимально вмешаться в текущий UX
  • Не добавляет новых концепций — пользователь сразу разберётся
  • ~80 LoC изменений в index.html
02
Presets library + CURATED
Сверху — лента предустановленных градиентов («CYBER», «ARCTIC», «FERRARI», «STEALTH»…). Тап на пресет загружает его в редактор, после этого можно докрутить. Свои пресеты сохраняются в localStorage.

Что нового

  • 8 встроенных пресетов в горизонтальной ленте
  • Тап на пресет → загружается в редактор. Можно подкрутить и заапплаить
  • «+ save current» → сохраняет текущий набор как user-пресет (localStorage). Своему пресету можно дать имя

Список встроенных

  • CYBER — cyan → green → yellow → red (default)
  • ARCTIC — холодная палитра до red
  • FERRARI — pure yellow → pure red
  • HEAT, STEALTH, FOREST, SUNSET, MONO
03
Step ↔ smooth toggle INTERPOLATION
Те же stops, но переключаемая интерполяция. STEP (как сейчас) — резкие скачки цвета при переходе через порог. SMOOTH — плавный lerp между stops. ESP сразу видит обоих режима, для firmware нужен новый branch в halo_color.
Interpolation mode
Step — firmware now
Smooth — proposed
saved · step
Step: at pct=53, color jumps from green to yellow at the 50% boundary. Visual feedback is binary — «спокойно / тревога».
Smooth: at pct=53, color is interpolated between green (50%) and yellow (70%) — slight yellowish tint, 15% of the way. Visual feedback is continuous — приятнее для глаза, но менее «алармно».

Что нового

  • Переключатель STEP / SMOOTH в правом углу заголовка
  • Side-by-side preview двух режимов на одном наборе stops
  • Mode сохраняется через cfg.gradient_mode на ESP (новое поле)

Firmware impact

  • halo_color() ветвится по новому полю gradient_mode
  • SMOOTH = существующая lerp-формула, +~20 LoC в emo2.cpp
  • Web preview уже умеет SMOOTH через CSS linear-gradient
  • Daemon: один extra ключ gradient_mode в cfg JSON
04
All-in-one master FULL
Объединение всех трёх: presets-rail сверху, refined editor с лочкой первого stop'а в центре, smooth/step toggle в action-row. Один экран — все возможности. Это то, что я предлагаю положить в основную страницу, если зайдёт.

Включено

  • ✓ Presets row + сохранение в localStorage
  • ✓ Lock первого stop на 0%
  • ✓ Pins прямо на gradient strip
  • ✓ Step / Smooth переключатель
  • ✓ Шкала 0/20/40/60/80/100 под полосой

Не включено (намеренно)

  • Eyedropper (брать цвет с экрана) — overkill для embedded usage
  • Color picker через HSV-popover — оставляем native <input type="color">
  • Numeric HSL/RGB ввод — пользователь работает с hex, этого хватает
