# EMO-style анимированные глаза для ESP32 + LVGL 9 + GFX Library

## Контекст проекта

Дисплей: 320×240 IPS/TFT  
Контроллер: ESP32  
Графический стек: LVGL 9 + Arduino GFX Library  
Референс: робот EMO от Living.AI (бирюзовые скруглённые прямоугольники-глаза на чёрном фоне)

```
Я делаю проект на ESP32 с дисплеем 320×240. Графический стек: LVGL 9 + Arduino GFX Library 
(автор moononournation). Хочу реализовать анимированного персонажа в стиле робота EMO от 
Living.AI — два светящихся бирюзовых глаза-прямоугольника со скруглёнными углами на чёрном 
фоне, которые постоянно живут: моргают, оглядываются, выражают эмоции.

ТРЕБОВАНИЯ К ВИЗУАЛУ:

Геометрия (в пикселях, для 320×240):
- eye_width = 66, eye_height = 70
- corner_radius = 22 (примерно треть от ширины)
- gap_between_eyes = 44
- center_y = 115 (чуть выше геометрического центра 120 — это важно для «милого» вида)
- left_eye_x = 92, right_eye_x = 202 (центр глаза)

Палитра (RGB565 для GFX Library):
- Фон:           #000000  →  0x0000
- Глаз основной: #5DD5D5  →  0x6E7A  (бирюзовый)
- Хайлайт:       #7DEEEE  →  0x77BD  (для верхней части/градиента)
- Тёмный край:   #3AB8B8  →  0x3DD7
- Акцент любви:  #FF5577  →  0xFAAE
- Акцент успеха: #3FB950  →  0x3DAA
- Акцент мысли:  #FFEB85  →  0xFF50
- Капли/грусть:  #58A6FF  →  0x5D3F

ТРЕБОВАНИЯ К АРХИТЕКТУРЕ:

1. State machine персонажа на C++. Состояния:
   neutral, happy, love, excited, wink, sad, angry, worried, shy, cry,
   sleep, blink, thinking, bored, focused, loading, success, error, dizzy, glitch
   
2. Каждое состояние — это набор параметров для двух «глаз»:
   struct EyeState {
     int16_t cx, cy;        // центр
     int16_t width, height; // размеры
     int16_t radius;        // скругление
     uint16_t color;        // RGB565
     int8_t  offset_x, offset_y; // саккада-смещение
     uint8_t shape_type;    // 0=rect, 1=arc_smile, 2=triangle_sad, 3=heart, 4=line_closed
   };
   
3. Анимация между состояниями — через линейную интерполяцию параметров за N мс. 
   Используй easing (ease-in-out cubic) для естественности.

4. Idle-поведение работает ВСЕГДА поверх текущего состояния:
   - blink каждые 3–6 секунд (рандом)
   - saccade (быстрый взгляд в сторону) каждые 8–15 секунд
   - breathing scale 98%↔102% непрерывно (sine, период 4 сек)
   - subpixel jitter ±1px (псевдослучайно)

5. Рендеринг через GFX Library Canvas (off-screen буфер) + DMA push на экран. 
   Это критично для плавности — прямая отрисовка на дисплей даст тиринг.
   
   Используй Arduino_Canvas как back-buffer 320×240×16бит = 150 КБ. Если не хватает RAM, 
   разбей на полосы по 80px (40 КБ) и пушай полосами.

6. LVGL 9 используй для оверлеев — статус-бар, иконки, текст-уведомления. Не пытайся 
   рисовать глаза через LVGL — оно медленнее, чем прямые draw-вызовы GFX Library для 
   такого юзкейса.

КЛЮЧЕВЫЕ ФОРМЫ ГЛАЗ ПО СОСТОЯНИЯМ:

- neutral: rounded_rect 66×70 r=22
- happy:   arc snippet (улыбающаяся дуга) — нарисуй через fillTriangle + fillCircle 
           по краям, или используй своё draw-функцию для дуги
- love:    сердечко — комбинация двух кругов + треугольник вниз
- excited: rounded_rect 78×90 r=27 (увеличенные) + маленькие звёздочки рядом
- wink:    левый — neutral, правый — горизонтальная линия (закрытый глаз)
- sad:     trapezoid с наклонёнными внутрь верхами (грустные брови)
- angry:   trapezoid с наклонёнными внутрь низами (нахмурен)
- blink:   плавно меняем height с 70 до 2 за 110 мс и обратно за 90 мс
- sleep:   две горизонтальные линии + анимированная буква Z над правым глазом

ЭФФЕКТ СВЕЧЕНИЯ (важно для премиум-вида):
Не просто закрашивай прямоугольник одним цветом. Делай два слоя:
1. Внешний rounded_rect основным цветом #5DD5D5
2. Внутренний меньший rounded_rect с цветом хайлайта #7DEEEE на 50% непрозрачности 
   (на ESP32 проще — просто более светлый цвет), смещённый вверх на 2-3 пикселя.
   
Это имитирует CRT-свечение и моментально делает глаза «живыми» вместо «нарисованных».

ЧТО Я ХОЧУ ОТ ТЕБЯ:

1. Полный скетч ESP32 (.ino) с включёнными библиотеками для типичной конфигурации 
   (ILI9341 или ST7789 на VSPI, GPIO по умолчанию для DevKitC).
2. Класс CharacterRenderer с публичными методами:
   - setEmotion(EmotionType e, uint16_t transition_ms = 300)
   - tick() — вызывается каждый кадр в loop()
   - и приватной state machine.
3. Реализацию хотя бы 6 ключевых эмоций: neutral, happy, sad, blink, sleep, love.
4. Работающий idle-loop с blink + saccade + breathing.
5. В комментариях покажи, как добавить новые эмоции по тому же паттерну.

Если что-то непонятно по железу — спрашивай, какой именно драйвер дисплея у меня 
(ILI9341/ST7789/ILI9488), как подключён. Не угадывай.

Начни с архитектурного плана, потом код.
```

## Архитектурная схема (для понимания)

```
┌─────────────────────────────────────────────────┐
│  loop() в .ino                                  │
│  └─ character.tick()                            │
│     ├─ stateMachine.update(millis())            │
│     │   ├─ idle: blink/saccade/breathing        │
│     │   └─ emotion: interpolate to target       │
│     ├─ renderer.drawFrame()                     │
│     │   ├─ canvas.fillScreen(BLACK)             │
│     │   ├─ drawEye(LEFT, state.left)            │
│     │   ├─ drawEye(RIGHT, state.right)          │
│     │   └─ drawOverlays(state.particles)        │
│     └─ canvas.flush() → DMA → дисплей           │
└─────────────────────────────────────────────────┘
                  ↑
                  │ внешние триггеры
                  │
   ┌──────────────┼──────────────┐
   │              │              │
WiFi/HTTP      MQTT          Sensors
(GitHub        (CI/CD       (MPU6050,
 webhook)       events)      кнопки)
```

## Бюджет производительности

- Canvas 320×240×2 байта = 153 КБ RAM. ESP32 имеет ~320 КБ — хватит, но впритык.
- Если станет тесно, используй PSRAM (модули с -WROVER) — там 4–8 МБ.
- Целевой FPS: 30. На ESP32 + DMA + SPI 40 МГц это реально для такого простого контента.
- Главное узкое место: не вычисления, а push на дисплей. DMA обязателен.
